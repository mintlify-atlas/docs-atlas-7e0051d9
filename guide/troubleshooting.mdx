## Task Management Issues

### Task Not Being Added

<AccordionGroup>
  <Accordion title="Symptom: Clicking 'Add' doesn't create a task" icon="circle-exclamation">
    **Possible causes:**
    
    1. **Empty task name**: The form field is required, but if JavaScript is disabled or you bypass validation, the backend has a safety check:
    
    ```python routes.py:15-22
    @app.route("/add", methods=["POST"])
    def add():
        name = request.form.get("task_name")
        if name:  # Only creates task if name exists
            task = Task(name=name)
            db.session.add(task)
            db.session.commit()
        return redirect(url_for("index"))
    ```
    
    **Solution**: Ensure you enter text in the input field before clicking Add.
    
    2. **Database connection issue**: The app might not be able to write to the database.
    
    **Solution**: Check your database connection and ensure the database file/server is accessible.
    
    3. **Form submission error**: The form might not be POSTing correctly.
    
    **Solution**: Check browser console for JavaScript errors. Ensure the form action points to `/add`:
    
    ```html index.html:13-16
    <form action="{{ url_for('add') }}" method="POST" class="board__form">
      <input type="text" name="task_name" 
             placeholder="What do you want to do?" required>
      <button type="submit" class="board__button">Add</button>
    </form>
    ```
  </Accordion>
  
  <Accordion title="Symptom: Task name is truncated" icon="scissors">
    **Cause**: The database column allows a maximum of 255 characters:
    
    ```python models.py:5
    name = db.Column(db.String(255), nullable=False)
    ```
    
    **Solution**: Keep task names under 255 characters. For longer descriptions, consider adding a separate notes field or use abbreviations.
  </Accordion>
</AccordionGroup>

### Task Won't Delete

<Warning>
If you see a 🔒 icon instead of a ❌ delete button, the task is locked and cannot be deleted through the web interface.
</Warning>

<Steps>
  <Step title="Check if the task is locked">
    Look for the lock icon (🔒) next to the task. Locked tasks have this icon instead of the delete button (❌).
    
    ```html index.html:23-27
    {% if not task.locked %}
      <a href="{{ url_for('delete', task_id=task.id) }}" 
         class="delete-btn" title="Delete">❌</a>
    {% else %}
      <span class="locked-icon" title="Locked task">🔒</span>
    {% endif %}
    ```
  </Step>
  
  <Step title="Verify the backend protection">
    Even if you try to delete via URL, the backend checks the lock status:
    
    ```python routes.py:32-38
    @app.route("/delete/<int:task_id>")
    def delete(task_id):
        task = Task.query.get(task_id)
        if task and not task.locked:  # Lock check here
            db.session.delete(task)
            db.session.commit()
        return redirect(url_for("index"))
    ```
  </Step>
  
  <Step title="Unlock the task if needed">
    To delete a locked task, you must first unlock it via database access:
    
    **SQLite:**
    ```sql
    UPDATE task SET locked = 0 WHERE id = <task_id>;
    ```
    
    **PostgreSQL:**
    ```sql
    UPDATE task SET locked = false WHERE id = <task_id>;
    ```
    
    See the [Task Locking guide](/guide/task-locking) for detailed instructions.
  </Step>
</Steps>

### Task Won't Toggle Between Complete/Incomplete

<AccordionGroup>
  <Accordion title="Symptom: Clicking checkbox does nothing" icon="square">
    **Possible causes:**
    
    1. **Database write failure**: The toggle route might not be able to commit changes.
    
    ```python routes.py:24-30
    @app.route("/toggle/<int:task_id>")
    def toggle(task_id):
        task = Task.query.get(task_id)
        if task:
            task.completed = not task.completed
            db.session.commit()  # Might fail here
        return redirect(url_for("index"))
    ```
    
    **Solution**: Check database permissions and ensure the app can write to the database file/server.
    
    2. **Task doesn't exist**: The task ID might be invalid.
    
    **Solution**: Refresh the page. If another user/process deleted the task, it won't exist anymore.
    
    3. **Caching issue**: Your browser might be showing stale data.
    
    **Solution**: Hard refresh the page (Ctrl+F5 or Cmd+Shift+R) or clear browser cache.
  </Accordion>
  
  <Accordion title="Symptom: Task moves but immediately reverts" icon="arrows-rotate">
    **Cause**: Database transaction is rolling back, possibly due to:
    - Database lock/conflict
    - Connection timeout
    - Insufficient permissions
    
    **Solution**: 
    1. Check application logs for database errors
    2. Verify database integrity
    3. Ensure no other processes are locking the database
    4. Check that SQLALCHEMY_DATABASE_URI is correctly configured
  </Accordion>
</AccordionGroup>

## Database Issues

### Database File Not Found (SQLite)

<Steps>
  <Step title="Check the database path">
    The database URI is configured in your application config. Check the SQLALCHEMY_DATABASE_URI setting.
  </Step>
  
  <Step title="Verify database initialization">
    The app creates tables on startup:
    
    ```python __init__.py:23-26
    with app.app_context():
        from . import routes
        from .models import Task
        db.create_all()
    ```
    
    This should create the database file if it doesn't exist.
  </Step>
  
  <Step title="Check file permissions">
    Ensure the application has write permissions to the directory where the SQLite file should be created.
    
    ```bash
    ls -la /path/to/database/directory
    chmod 755 /path/to/database/directory  # If needed
    ```
  </Step>
  
  <Step title="Manually create the database">
    If automatic creation fails, you can initialize manually:
    
    ```python
    from app import create_app, db
    app = create_app()
    with app.app_context():
        db.create_all()
    ```
  </Step>
</Steps>

### Connection Refused (PostgreSQL/MySQL)

<Note>
The app displays the current database engine in the footer, helping you verify which database system is in use.

```html index.html:50-54
<footer>
  <div class="footer-engine">
    Database Engine: {{ engine }}
  </div>
</footer>
```
</Note>

<Tabs>
  <Tab title="PostgreSQL">
    **Common issues:**
    
    1. **Service not running**
       ```bash
       # Check status
       sudo systemctl status postgresql
       
       # Start if needed
       sudo systemctl start postgresql
       ```
    
    2. **Wrong credentials**
       - Verify SQLALCHEMY_DATABASE_URI format:
         ```
         postgresql://username:password@localhost:5432/dbname
         ```
       - Test connection manually:
         ```bash
         psql -U username -d dbname -h localhost
         ```
    
    3. **Firewall blocking port 5432**
       ```bash
       sudo ufw allow 5432/tcp
       ```
  </Tab>
  
  <Tab title="MySQL">
    **Common issues:**
    
    1. **Service not running**
       ```bash
       # Check status
       sudo systemctl status mysql
       
       # Start if needed
       sudo systemctl start mysql
       ```
    
    2. **Wrong connection URI**
       - Format should be:
         ```
         mysql+pymysql://username:password@localhost:3306/dbname
         ```
       - Ensure you have the correct driver installed:
         ```bash
         pip install pymysql
         ```
    
    3. **Authentication plugin issue**
       - MySQL 8+ uses caching_sha2_password by default
       - You may need to change the user's authentication method:
         ```sql
         ALTER USER 'username'@'localhost' 
         IDENTIFIED WITH mysql_native_password BY 'password';
         ```
  </Tab>
</Tabs>

### Migration/Schema Issues

<AccordionGroup>
  <Accordion title="Column doesn't exist" icon="table">
    **Symptom**: Error like `no such column: task.locked` or `column "locked" does not exist`
    
    **Cause**: Your database schema is outdated and doesn't include the `locked` column.
    
    **Solution**: The `locked` field was added to the Task model:
    
    ```python models.py:7
    locked = db.Column(db.Boolean, default=False)
    ```
    
    To update your database:
    
    **Option 1 - Fresh start (destroys data):**
    ```python
    from app import create_app, db
    app = create_app()
    with app.app_context():
        db.drop_all()  # Deletes all tables
        db.create_all()  # Recreates with current schema
    ```
    
    **Option 2 - Manual migration (preserves data):**
    
    For SQLite:
    ```sql
    ALTER TABLE task ADD COLUMN locked BOOLEAN DEFAULT 0;
    ```
    
    For PostgreSQL:
    ```sql
    ALTER TABLE task ADD COLUMN locked BOOLEAN DEFAULT false;
    ```
  </Accordion>
  
  <Accordion title="Table doesn't exist" icon="database">
    **Symptom**: Error like `no such table: task`
    
    **Cause**: Database hasn't been initialized.
    
    **Solution**: Run the initialization code:
    
    ```python
    from app import create_app, db
    app = create_app()
    with app.app_context():
        db.create_all()
    ```
    
    This should happen automatically when the app starts (see app/__init__.py:23-26), but may fail if there are permission issues or path problems.
  </Accordion>
</AccordionGroup>

## Display and UI Issues

### Page Doesn't Load or Shows 500 Error

<Steps>
  <Step title="Check Flask logs">
    Look for error messages in the terminal where you're running the Flask app. Common errors include:
    - Import errors
    - Database connection failures
    - Template rendering issues
  </Step>
  
  <Step title="Verify all dependencies are installed">
    ```bash
    pip install flask flask-sqlalchemy
    ```
  </Step>
  
  <Step title="Check the index route">
    The home page route must successfully query tasks:
    
    ```python routes.py:6-12
    @app.route("/")
    def index():
        tasks = Task.query.all()
        pending = [t for t in tasks if not t.completed]
        completed = [t for t in tasks if t.completed]
        style = app.config.get("style", {})
        return render_template("index.html", pending=pending, 
                             completed=completed, style=style)
    ```
    
    If `Task.query.all()` fails, you'll get a 500 error.
  </Step>
  
  <Step title="Test database connection">
    ```python
    from app import create_app, db
    from app.models import Task
    
    app = create_app()
    with app.app_context():
        try:
            count = Task.query.count()
            print(f"Database OK: {count} tasks found")
        except Exception as e:
            print(f"Database error: {e}")
    ```
  </Step>
</Steps>

### Styling Issues

<AccordionGroup>
  <Accordion title="No styles applied" icon="paintbrush">
    **Possible causes:**
    
    1. **CSS file not loaded**: Check browser console for 404 errors
    2. **Static files path incorrect**: Verify your Flask static folder configuration
    3. **Browser cache**: Hard refresh (Ctrl+F5) to clear cached styles
    
    **Note**: The app supports custom styling via config:
    
    ```python routes.py:11-12
    style = app.config.get("style", {})
    return render_template("index.html", ..., style=style)
    ```

  </Accordion>
  
  <Accordion title="Icons not displaying correctly" icon="icons">
    **Symptom**: Emoji icons (❌, 🔒, ☐, ☑) appear as squares or boxes
    
    **Cause**: Font or character encoding issue
    
    **Solution**: 
    1. Ensure your HTML has proper charset declaration
    2. Use a modern browser that supports emoji rendering
    3. Consider replacing emojis with icon fonts (Font Awesome, Material Icons) for better cross-platform support
  </Accordion>
</AccordionGroup>

### Tasks Not Appearing in Correct Section

<Note>
Tasks are separated into "Pending" and "Completed" sections based on their `completed` status.
</Note>

**How the separation works:**

```python routes.py:8-10
tasks = Task.query.all()
pending = [t for t in tasks if not t.completed]
completed = [t for t in tasks if t.completed]
```

**Troubleshooting:**

1. **Check the task's completed value in the database:**
   ```sql
   SELECT id, name, completed FROM task;
   ```

2. **Verify the template is rendering both lists:**
   ```html index.html
   <!-- Pending tasks section -->
   <div class="board__items">
     {% for task in pending %}
     ...
     {% endfor %}
   </div>
   
   <!-- Completed tasks section -->
   <h3 class="board__subtitle">Completed</h3>
   <div class="board__items">
     {% for task in completed %}
     ...
     {% endfor %}
   </div>
   ```

3. **Try toggling the task**: Click the checkbox to flip the status and see if it moves correctly

## Performance Issues

### Slow Page Loads with Many Tasks

<Warning>
The current implementation loads ALL tasks on every page load:

```python routes.py:8
tasks = Task.query.all()
```

With thousands of tasks, this can become slow.
</Warning>

**Solutions:**

<Tabs>
  <Tab title="Add Pagination">
    Limit the number of tasks displayed per page:
    
    ```python
    from flask import request
    
    @app.route("/")
    def index():
        page = request.args.get('page', 1, type=int)
        per_page = 20
        
        tasks = Task.query.paginate(page=page, per_page=per_page)
        pending = [t for t in tasks.items if not t.completed]
        completed = [t for t in tasks.items if t.completed]
        
        return render_template("index.html", pending=pending, 
                             completed=completed, 
                             pagination=tasks)
    ```
  </Tab>
  
  <Tab title="Filter Completed Tasks">
    Only show pending tasks by default:
    
    ```python
    @app.route("/")
    def index():
        pending = Task.query.filter_by(completed=False).all()
        completed = Task.query.filter_by(completed=True).limit(10).all()
        
        return render_template("index.html", pending=pending, 
                             completed=completed)
    ```
  </Tab>
  
  <Tab title="Add Database Indexing">
    Create an index on the completed column:
    
    ```python models.py
    class Task(db.Model):
        id = db.Column(db.Integer, primary_key=True)
        name = db.Column(db.String(255), nullable=False)
        completed = db.Column(db.Boolean, default=False, index=True)
        locked = db.Column(db.Boolean, default=False)
    ```
    
    Then recreate your database or add the index manually:
    ```sql
    CREATE INDEX idx_task_completed ON task(completed);
    ```
  </Tab>
</Tabs>

## Common Error Messages

| Error | Location | Solution |
|-------|----------|----------|
| `no such table: task` | Database | Run `db.create_all()` in app context |
| `no such column: task.locked` | Database | Add locked column via migration |
| `UNIQUE constraint failed` | Routes | Trying to add duplicate ID (shouldn't happen with auto-increment) |
| `404 Not Found` | Routes | Check URL matches route definitions |
| `405 Method Not Allowed` | Routes | Ensure POST requests go to `/add`, GET to others |
| `OperationalError` | Database | Database file locked or connection failed |
