## Overview

MariaDB is a MySQL-compatible, production-ready relational database. It offers excellent performance, reliability, and scalability for multi-user environments.

<Note>
  MariaDB is recommended for production deployments where you need robust concurrent access and transaction support.
</Note>

## Database Setup

### Create Database and User

Connect to MariaDB as root:

```bash
mysql -u root -p
```

Create the database and user:

```sql
-- Create database
CREATE DATABASE todo_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Create user with password
CREATE USER 'todouser'@'localhost' IDENTIFIED BY 'your_secure_password';

-- Grant privileges
GRANT ALL PRIVILEGES ON todo_db.* TO 'todouser'@'localhost';

-- Flush privileges
FLUSH PRIVILEGES;

-- Verify
SHOW GRANTS FOR 'todouser'@'localhost';

-- Exit
EXIT;
```

<Warning>
  Replace `your_secure_password` with a strong password. Never use default passwords in production!
</Warning>

## Configuration

### Database Config

Update your `db-config.yaml` to use MariaDB:

```yaml db-config.yaml
engine: mariadb

sqlite:
  database: /full/path/to/todo.db

mariadb:
  user: todouser
  password: your_secure_password
  host: localhost
  port: 3306
  database: todo_db

postgresql:
  user: postgres
  password: pgpass
  host: localhost
  port: 5432
  database: todo_db
```

### Connection URI

The application constructs the MariaDB connection string:

```python app/db_utils.py
elif engine == "mariadb":
    mdb = cfg["mariadb"]
    return f"mysql+pymysql://{mdb['user']}:{mdb['password']}@{mdb['host']}:{mdb['port']}/{mdb['database']}"
```

This generates a URI like:
```
mysql+pymysql://todouser:your_secure_password@localhost:3306/todo_db
```

### Configuration Options

| Parameter | Description | Default | Required |
|-----------|-------------|---------|----------|
| `user` | Database username | - | Yes |
| `password` | Database password | - | Yes |
| `host` | Database server hostname | `localhost` | Yes |
| `port` | Database server port | `3306` | Yes |
| `database` | Database name | - | Yes |

## Database Initialization

### Automatic Initialization

SQLAlchemy automatically creates tables when the application starts:

```python app/__init__.py
with app.app_context():
    from . import routes
    from .models import Task
    db.create_all()  # Creates tables automatically
```

### Manual Initialization

Use the provided initialization script for manual setup:

```bash scripts/init_db_mariadb.sh
#!/bin/bash

DB_NAME="todo_db"
USER="your_user"
PASSWORD="your_password"
HOST="localhost"
PORT="3306"

echo "Initializing MariaDB schema..."

mysql -u"$USER" -p"$PASSWORD" -h "$HOST" -P "$PORT" "$DB_NAME" <<EOF
CREATE TABLE IF NOT EXISTS task (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    completed BOOLEAN DEFAULT FALSE,
    locked BOOLEAN DEFAULT FALSE
);
EOF

if [ $? -ne 0 ]; then
  echo "Failed!"
  exit 1
else
  echo "MariaDB schema initialized."
  exit 0
fi
```

## Database Schema

The MariaDB schema matches the SQLAlchemy model:

<CodeGroup>

```sql Raw SQL Schema
CREATE TABLE IF NOT EXISTS task (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    completed BOOLEAN DEFAULT FALSE,
    locked BOOLEAN DEFAULT FALSE
);
```

```python SQLAlchemy Model
from . import db

class Task(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(255), nullable=False)
    completed = db.Column(db.Boolean, default=False)
    locked = db.Column(db.Boolean, default=False)
```

</CodeGroup>

### Field Mappings

| SQLAlchemy Type | MariaDB Type | Notes |
|-----------------|--------------|-------|
| `db.Integer` (PK) | `INT AUTO_INCREMENT` | Auto-incrementing primary key |
| `db.String(255)` | `VARCHAR(255)` | Variable-length string |
| `db.Boolean` | `BOOLEAN` | Stored as TINYINT(1) |

## Performance Optimization

### Indexing

Add indexes for frequently queried columns:

```sql
-- Index on completed status for filtering
CREATE INDEX idx_completed ON task(completed);

-- Index on locked status
CREATE INDEX idx_locked ON task(locked);

-- Composite index for common queries
CREATE INDEX idx_completed_locked ON task(completed, locked);
```

### Connection Pooling

Configure SQLAlchemy connection pool settings:

```python
# In app/__init__.py or config
app.config['SQLALCHEMY_POOL_SIZE'] = 10
app.config['SQLALCHEMY_POOL_RECYCLE'] = 3600
app.config['SQLALCHEMY_POOL_TIMEOUT'] = 30
app.config['SQLALCHEMY_MAX_OVERFLOW'] = 20
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection refused">
    **Error**: `Can't connect to MySQL server on 'localhost'`
    
    **Solution**:
    - Verify MariaDB is running: `sudo systemctl status mariadb`
    - Start MariaDB: `sudo systemctl start mariadb`
    - Check port 3306 is open: `sudo netstat -tlnp | grep 3306`
    - Verify firewall rules allow connection
  </Accordion>
  
  <Accordion title="Access denied">
    **Error**: `Access denied for user 'todouser'@'localhost'`
    
    **Solution**:
    - Verify credentials in `db-config.yaml` are correct
    - Check user exists: `SELECT User, Host FROM mysql.user;`
    - Verify grants: `SHOW GRANTS FOR 'todouser'@'localhost';`
    - Recreate user if needed (see Database Setup section)
  </Accordion>
  
  <Accordion title="Unknown database">
    **Error**: `Unknown database 'todo_db'`
    
    **Solution**:
    - Create the database: `CREATE DATABASE todo_db;`
    - Verify databases: `SHOW DATABASES;`
    - Check `database` field in `db-config.yaml` matches
  </Accordion>
  
  <Accordion title="Table doesn't exist">
    **Error**: `Table 'todo_db.task' doesn't exist`
    
    **Solution**:
    - Run initialization script: `./scripts/init_db_mariadb.sh`
    - Or let SQLAlchemy create it by starting the app
    - Verify with: `SHOW TABLES;`
  </Accordion>
  
  <Accordion title="PyMySQL not installed">
    **Error**: `ModuleNotFoundError: No module named 'pymysql'`
    
    **Solution**:
    ```bash
    pip install pymysql
    ```
  </Accordion>
</AccordionGroup>

## Security Best Practices

<Card title="Security Checklist" icon="shield">
  - Use strong, unique passwords for database users
  - Limit user privileges (grant only what's needed)
  - Keep MariaDB updated to latest stable version
  - Use SSL/TLS for remote connections
  - Disable remote root access
  - Regular backup strategy
  - Monitor slow query log for optimization
  - Use environment variables for credentials, not hardcoded values
</Card>

## Remote Access

To connect from a remote host:

```sql
-- Create user for remote access
CREATE USER 'todouser'@'%' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON todo_db.* TO 'todouser'@'%';
FLUSH PRIVILEGES;
```

Update MariaDB config to bind to all interfaces:

```ini /etc/mysql/mariadb.conf.d/50-server.cnf
[mysqld]
bind-address = 0.0.0.0
```

Restart MariaDB:
```bash
sudo systemctl restart mariadb
```

<Warning>
  Remote access should be restricted by firewall rules. Only allow connections from trusted IP addresses.
</Warning>
