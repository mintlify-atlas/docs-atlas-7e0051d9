## Overview

The Flask ToDo App uses two YAML configuration files:
- `config.yaml` - Main application configuration
- `db-config.yaml` - Database connection settings

Configuration is loaded via `config.py:5` using the `load_app_config()` function.

## config.yaml

The main application configuration file that defines database config path, styling options, and Flask server settings.

### Schema

<ParamField path="database_config_path" type="string" required>
  Absolute path to the database configuration file (`db-config.yaml`)
  
  **Example:** `/full/path/to/db-config.yaml`
</ParamField>

<ParamField path="style" type="object">
  Custom styling configuration for the web interface (optional)
  
  <Expandable title="Style Properties">
    <ParamField path="style.background_color" type="string">
      Background color of the page (hex color code)
      
      **Example:** `#1a1a1a`
    </ParamField>
    
    <ParamField path="style.text_color" type="string">
      Primary text color (hex color code)
      
      **Example:** `#2a750f`
    </ParamField>
    
    <ParamField path="style.button_color" type="string">
      Button background color (hex color code)
      
      **Example:** `#4cc121`
    </ParamField>
    
    <ParamField path="style.button_text_color" type="string">
      Button text color (hex color code)
      
      **Example:** `#ffffff`
    </ParamField>
    
    <ParamField path="style.completed_text_color" type="string">
      Text color for completed tasks (hex color code)
      
      **Example:** `#6c757d`
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="flask" type="object">
  Flask server configuration settings
  
  <Expandable title="Flask Properties">
    <ParamField path="flask.port" type="integer" default="8080">
      Port number for the Flask development server
      
      **Example:** `8080`
    </ParamField>
    
    <ParamField path="flask.debug" type="boolean" default="false">
      Enable Flask debug mode (auto-reload, detailed errors)
      
      **Values:** `true` or `false`
    </ParamField>
  </Expandable>
</ParamField>

## db-config.yaml

Database connection configuration supporting SQLite, MariaDB, and PostgreSQL.

**Implementation:** Database URI construction handled by `db_utils.py:3`

  Database engine type
  
  **Allowed values:** `sqlite`, `mariadb`, `postgresql`

## Configuration Loading

The `load_app_config()` function in `config.py` handles the configuration loading process:

```python config.py
def load_app_config():
    with open("config.yaml", "r") as f:
        cfg = yaml.safe_load(f)

    db_config_path = cfg["database_config_path"]

    if not os.path.isfile(db_config_path):
        raise FileNotFoundError(f"Database config file not found: {db_config_path}")

    with open(db_config_path, "r") as f:
        db_cfg = yaml.safe_load(f)

    db_uri = construct_db_uri(db_cfg)
    return {
        "SQLALCHEMY_DATABASE_URI": db_uri,
        "style": cfg.get("style", {}),
        "flask": cfg.get("flask", {"port": 8080, "debug": False})
    }
```

### Loading Process

<Steps>
  <Step title="Load config.yaml">
    Reads and parses the main configuration file using `yaml.safe_load()`
  </Step>
  
  <Step title="Validate Database Config Path">
    Checks that the `database_config_path` points to an existing file, raises `FileNotFoundError` if missing
  </Step>
  
  <Step title="Load db-config.yaml">
    Reads and parses the database configuration file
  </Step>
  
  <Step title="Construct Database URI">
    Calls `construct_db_uri()` from `db_utils.py` to build the SQLAlchemy connection string
  </Step>
  
  <Step title="Return Configuration Dictionary">
    Returns a dictionary with SQLAlchemy URI, style settings, and Flask settings
  </Step>
</Steps>

## Database URI Construction

The `construct_db_uri()` function in `db_utils.py` builds database connection strings:

```python db_utils.py
def construct_db_uri(cfg):
    global engine_type
    engine_type = cfg.get("engine")
    engine = engine_type

    if engine == "sqlite":
        sqlite_cfg = cfg["sqlite"]
        return f"sqlite:///{sqlite_cfg['database']}"

    elif engine == "mariadb":
        mdb = cfg["mariadb"]
        return f"mysql+pymysql://{mdb['user']}:{mdb['password']}@{mdb['host']}:{mdb['port']}/{mdb['database']}"

    elif engine == "postgresql":
        pg = cfg["postgresql"]
        return f"postgresql+psycopg2://{pg['user']}:{pg['password']}@{pg['host']}:{pg['port']}/{pg['database']}"

    else:
        raise ValueError(f"Unsupported engine: {engine}")
```

## Configuration Best Practices

<AccordionGroup>
  <Accordion title="Use Absolute Paths">
    Always use absolute paths for file references:
    
    ```yaml
    # Good
    database_config_path: "/home/user/app/db-config.yaml"
    sqlite:
      database: /var/lib/todo/todo.db
    
    # Bad
    database_config_path: "./db-config.yaml"
    sqlite:
      database: todo.db
    ```
  </Accordion>
  
  <Accordion title="Secure Sensitive Data">
    Never commit passwords or sensitive data to version control:
    
    - Add `db-config.yaml` to `.gitignore`
    - Use environment variables for production passwords
    - Consider using secrets management tools
  </Accordion>
  
  <Accordion title="Configure All Database Sections">
    Even if you're only using one database engine, configure all sections in `db-config.yaml`:
    
    - Makes switching databases easier
    - Provides clear documentation of options
    - Follows the template structure
  </Accordion>
  
  <Accordion title="Production vs Development">
    Use different configurations for different environments:
    
    ```yaml Development
    flask:
      port: 5000
      debug: true  # Enable debug mode
    ```
    
    ```yaml Production
    flask:
      port: 8080
      debug: false  # Disable debug mode
    ```
  </Accordion>
</AccordionGroup>

<Warning>
  The `engine` field in `db-config.yaml` must match one of the supported values: `sqlite`, `mariadb`, or `postgresql`. Any other value will raise a `ValueError`.
</Warning>

<Note>
  The style configuration is entirely optional. If omitted, the application will use default styling defined in the HTML templates.
</Note>
